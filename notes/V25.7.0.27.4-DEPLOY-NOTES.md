# v25.7.0.27.4 — Clean tactic page · all visualizations slide out from right

Final fix release for the tactic-page rendering question. Every
tactic page now renders identically to Initial Access — header +
executive takeaway + "X TECHNIQUES" toggle, nothing else above
the toggle. No inline visualization content. No "Interactive
content for other roles" affordance card. All tactic-attached
visualizations open via right-side slide-out, same way technique
animations do.

This release supersedes the .27.1 / .27.2 / .27.3 sequence and
becomes the canonical state for tactic-page rendering. Going
forward: every detail panel on the Framework page uses the same
slide-out interaction model with Section-based interior. One
rule, one pattern, no exceptions.

## What you'll see when this lands

**Every tactic page** (Reconnaissance, Initial Access, Defense
Evasion, Positioning, Execution, Resource Development, Monetization)
renders the same shape:

1. Tactic header — TA00xx code + tactic title
2. Description block (left col) + Executive Takeaway (right col)
3. **"X TECHNIQUES AND SUB-TECHNIQUES" toggle** (collapsed by
   default, click `↓` chevron to expand)

Nothing between the executive takeaway and the toggle. No
inline timeline, no inline two-views, no inline kill-chain
grid, no "Interactive content for other roles" card. Clean.

**When the toggle is expanded:**

- For tactics with attached visualizations (Reconnaissance,
  Positioning, Defense Evasion): visualization trigger cards
  appear at the top of the expanded area, then the technique
  cards grid below. Each viz card shows its title + subtitle +
  `→` arrow. Click → opens the right-side slide-out with the
  visualization rendered inside (Section-based interior matching
  F1067 / F1073).
- For tactics without visualizations (Initial Access, Resource
  Development, etc.): the technique cards grid renders directly,
  no viz cards above.
- Technique cards behave exactly as before — click → opens the
  right-side technique slide-out.

**For users whose role is excluded from a tactic's
visualizations:** the viz trigger cards do not render at all.
The user sees the same clean tactic page as before, with just
the technique cards inside the toggle. The "Interactive content
for other roles" affordance card that previously rendered in
this scenario is gone.

## Pedagogical insight (OBS-027 — methodological)

**The tactic page is a top-of-funnel. Detail content is a
slide-out. Period.**

The Framework page has two surfaces: the top-of-funnel (the
tactic list with its description and executive takeaway) and the
detail content (everything else). The role of the top-of-funnel
is orientation — the trainee scans the seven tactics, reads the
takeaways, decides where to dive in. The role of the detail
content is depth — full description, role gating, animations,
visualizations, cross-references to scenarios. Mixing the two
surfaces (inline visualizations rendering in the top-of-funnel
column) breaks the orientation function: the tactic page
becomes long, dense, and hard to scan, and the trainee loses
the "I'm picking which tactic to study" frame.

The fix is structural separation. Top-of-funnel = clean, scannable,
short. Detail content = slide-out from the right, focused, full
real estate. The visualization cards inside the techniques toggle
are the bridge: they live in the same "click-to-explore" affordance
as technique cards, communicating that they're peers with the
techniques in terms of "I'm a thing you can dive into" semantics.

The earlier coexistence of inline + slide-out for adjacent content
types was a release-sequencing artifact (visualizations shipped in
v25.7.0.3-.4 with inline rendering before the technique slide-out
was even authored in v25.7.0.8). This release reconciles them at
the architecture level rather than at any individual surface
level.

## Four-lens block (OBS-018)

- **Teller:** Tactic pages are dramatically cleaner — Reconnaissance,
  Positioning, Defense Evasion now match the Initial Access
  shape that tellers already navigate confidently. The

  visualization is authored for the analyst lens before

- **SOC:** Pure client-side changes to two files —
  `Framework.jsx` (refactor) and `TacticVisualizationSidebar.jsx`
  (restored from v25.7.0.27.2 with Section interior). Backend
  untouched, schema untouched, seed untouched, graph queries
  untouched. Risk surface limited to these two files. The
  underlying visualization components (`PositioningTimeline`,
  `PositioningTwoViews`, `KillChainGrid`,
  `DefenseEvasionTwoViews`) are unchanged.

- **Executive:** Demo flow improves substantially. Walking a
  CRO at NCB / JNCB / Scotia / CIBC / Sagicor through the
  Framework page, every tactic page now reads with the same
  rhythm: orient at the top (description + takeaway), expand
  the toggle, browse the cards, click any card (viz or
  technique) to dive into the detail panel. Consistency reads
  as discipline; the prospect doesn't notice "wait, why does
  this tactic look different than that one."

## What ships in this release

### Modified component

`client/src/pages/Framework.jsx` — six specific changes:

1. **New import:** `TacticVisualizationSidebar`
2. **Removed import:** `VisualizationRenderer` (no longer used in
   this file; moved to `TacticVisualizationSidebar.jsx`)
3. **Removed dead state:** `VIZ_DEFAULTS`, `isVizDefaultExpanded`,
   `vizToggleOverrides`, `isVizExpanded`, `toggleViz` — the
   inline expand-collapse machinery is gone with the inline
   rendering it backed.
4. **New state:** `vizSidebar` (`{ open, viz }`) +
   `openVizSidebar` / `closeVizSidebar` handlers, mirroring
   the existing technique sidebar state.
5. **Removed two inline rendering blocks:**
   - The "Viz exists for OTHER roles" branch (the
     `tacticBodyOtherRoleCard` + chips block — "Interactive
     content for other roles") — replaced with the same shape
     as the no-viz fallback (header → toggle → techniques)
   - The visible-viz inline expand-collapse block (the
     `visibleViz.map(...)` rendering with conditional
     `<VisualizationRenderer>`) — replaced with a comment
     pointing at the new location of the viz cards (inside
     the toggle-expanded techniques area)
6. **New rendering inside the toggle-expanded area:**
   visualization trigger cards render at the top of the
   `expandedTechniques.has(t.id)` block, above the
   `<TechniquesTree>`. Each card shows title + subtitle + `→`
   arrow. Click handler calls `openVizSidebar(viz)`.
7. **New page-level instance:** `<TacticVisualizationSidebar>`
   added next to `<TechniqueDetailSidebar>`, both render at
   page level so they overlay the entire Framework page when
   open.
