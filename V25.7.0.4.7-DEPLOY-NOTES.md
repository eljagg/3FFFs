# v25.7.0.4.7 — Unified F3 tactic layout

Restructures the expanded-tactic body into a single consistent shape that applies to ALL seven F3 tactics, regardless of which (or how many) visualizations they have attached.

## The new structure

```
┌─ Description ─────────────┬─ Executive Takeaway ──────────┐
│ Adversaries may collect... │ The tactic MITRE added...    │
│ (left half)                │ (right half)                  │
└────────────────────────────┴───────────────────────────────┘
[ Visualization 1 — full page-content-width                  ]
[ Visualization 2 — full page-content-width   (if present)   ]
[ ... etc                                                    ]
[ Techniques — collapsed by default, click to expand         ]
```

## What changed

Previously: the description text rendered in the LEFT column of a two-column grid, with visualizations squeezed into the RIGHT column at half-width. Tactics with short descriptions but tall visualizations had ~25 lines of empty dark space below the text where the right column's viz was still rendering.

Now:
- Header section is a balanced two-column grid: description left, takeaway right. Both prose blocks of similar length, no dead space.
- Visualizations are no longer squeezed into a half-column — they ALL render full page-content-width below the header, stacked sequentially.
- The previous "compact" vs "wide" visualization split is gone. Every viz gets full width.

## Applies to all F3 tactics

This is a Framework.jsx change — it affects every tactic in the F3 list:
- Reconnaissance (TA0043) — kill chain grid now renders at full 1500px width
- Resource Development (TA0042) — header section + collapsed techniques
- Initial Access (TA0001) — header section + collapsed techniques
- Defense Evasion (TA0005) — header section + collapsed techniques
- Positioning (FA0001) — header section + timeline (full-width) + Two Views (full-width) + collapsed techniques
- Execution (TA0002) — header section + collapsed techniques
- Monetization (FA0002) — header section + collapsed techniques

When future v25.7.0.x releases attach visualizations to the other tactics, they'll automatically use this same layout.

## Files changed

- `client/src/pages/Framework.jsx` — restructured the expanded-tactic body, added tacticHeaderTwoCol/tacticHeaderCol styles, removed the compact/wide split

`Page.jsx` and `PositioningTwoViews.jsx` carry forward unchanged from v25.7.0.4.6.
