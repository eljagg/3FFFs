# v24.2 Deployment Notes — confidence slider discovery treatment

**One file change.** Pure UX pop on the confidence slider so a first-time
user notices it on landing, without becoming noise for returning users.

## What changes visually

1. **Accent left-stripe** — 3px orange bar down the left edge of the
   confidence card. Same visual pattern as severity stripes on signal items,
   so the language is familiar; in accent colour it reads as "this is a new
   kind of thing" without being alarming.

2. **Lifted background** — card now uses `--paper-hi` instead of `--paper-dim`.
   On dark mode this is the lighter tone (used for elevated surfaces), so the
   confidence card visually sits "above" the surrounding stage panel rather
   than blending into it. Reads as: this is its own moment.

3. **"NEW" pill** — small accent-on-white pill next to the prompt label,
   monospaced, 9px caps. **Only shown the very first time per browser session**,
   and only until the user picks any confidence value. After that, the pill
   never appears again in that session (across all scenarios) — so this is
   genuinely a "discover the feature" cue, not persistent decoration.

4. **One-time mount glow** — when the first confidence card of the session
   mounts, a soft accent-coloured glow fades in (~0.6s) and out (~1s). Total
   animation duration ~1.6s. Returning users (anyone who has used the slider
   even once this session) skip the glow entirely.

The waiting-for-choice border pulse from v24.1 is preserved — it's still the
mechanism that flags "you need to pick this before answering."

## How "first time per session" works

A module-level boolean (`SEEN_THIS_SESSION`) flips to `true` the first time
any `ConfidenceSlider` component mounts. Subsequent mounts in the same
session see the flag is set and skip the discovery treatment.

The boolean resets when the page is hard-refreshed or the browser tab closes
— no localStorage. This is intentional: a user who comes back tomorrow gets
a gentle reminder; a user who churns through five scenarios in one sitting
sees the discovery treatment exactly once.

If you'd rather make this persistent across sessions (so a returning user
never sees the NEW pill or glow ever again), let me know and I'll add a
`localStorage` key. Default behaviour is "fresh per session" because over-
suppression is worse than mild repetition for a feature that's the cornerstone
of the v24 interactivity push.

## What to test

1. **Hard-refresh the app** (Ctrl+Shift+R), open any scenario.
2. The confidence card should fade in with a soft orange glow around it
   and a small "NEW" pill next to the prompt. Glow lasts ~1.5s and fades.
3. Pick a confidence value — the "NEW" pill disappears.
4. Navigate to another scenario. Confidence card on the new scenario shows
   the accent stripe and lifted background, but **no NEW pill and no glow** —
   you've already seen it this session.
5. Hard-refresh — the discovery treatment comes back on the next first
   confidence card you encounter.
