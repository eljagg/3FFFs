# v25.1.1 Deployment Notes — ConfidenceSlider visual fix (HOTFIX)

**Single-file hotfix.** No migration. No schema change. No server change.
The ConfidenceSlider was rendering with the v24.1 plain styling on SC010
(and arguably on every scenario in dark mode) because the v24.2 visual
treatment used `--paper-hi` for its background — the same tone the stage
panel itself uses, so the slider visually merged into the panel and the
"card-on-a-card" lift disappeared.

## What changed (and why)

**Root cause:** The slider sits *inside* a stage panel whose background is
`--paper-hi`. The v24.2 slider also used `--paper-hi`. Same tone, no
contrast, no perceptible lift. The 3px accent left-stripe was technically
present but read as "card edge" rather than "accent stripe" because of the
flush placement against the panel.

**Fix — three always-on changes:**
1. **Slider background → `--paper-dim`** (was `--paper-hi`). The slider now
   sits *darker* than the panel it lives in, reading as a recessed
   sub-panel — the visual "card-on-a-card" hierarchy works at every nesting
   level, every theme.
2. **Accent left-stripe → 4px** (was 3px). Unmistakable as an accent
   element rather than a card edge.
3. **"How confident are you?" label → 11px / weight 700 / `--ink`** (was
   10px / 600 / `--ink-soft`). Reads as a peer section header alongside
   "YOUR DECISION" and "SIGNALS OBSERVED".

Plus two minor refinements:
- Dot border bumped from 1.5px → 2px so unselected dots stay crisp on the
  now-darker `--paper-dim` background.
- Added a subtle `inset 0 1px 0 rgba(255,255,255,0.04)` highlight for depth.

**The discovery treatment is unchanged.** The "NEW" pill and the soft
accent glow on first-mount-per-session still fire exactly as designed in
v24.2 — first slider you encounter in a new browser session gets the
discovery flourish, every subsequent slider in the same session is quiet.
That mechanism was never broken; it just looked muted because the
underlying card was invisible.

4. **Toggle to light mode** (sun/moon icon top right). The slider should

## Why hard-refresh (Ctrl+Shift+R) is needed for the discovery treatment

The `SEEN_THIS_SESSION` flag is module-level — it lives as long as the
JavaScript module is in memory. A normal page navigation in the SPA
preserves the module. A hard refresh re-evaluates the module, which resets
the flag, which means the next slider you see gets the discovery treatment
again. This is by design (and unchanged from v24.2): we want the discovery
cue to feel fresh once per session, not nagging on every scenario.

If you want to verify the fix landed without bothering with the discovery
treatment, just open SC010 — the always-on visual identity (background,
stripe, label) will be visible regardless of whether the discovery pulse
fires.

## Rollback

`git revert HEAD && git push` reverts the file to v24.2 styling. No graph

- The slider's *behavior* is unchanged. Locking-on-answer, the value-state

  v24.2.
- The discovery treatment timing is unchanged. Still ~1.6s glow, still
  fires on first mount per session.
