# v25.4 Deployment Notes — Frameworks page (start of Phase 4)

**The dedicated Frameworks module is now live.** Top-nav entry called
"Frameworks" (plural) sits next to the existing "Framework" (singular,
F3 fraud) entry. The page surfaces the AASE/CBEST/TIBER-EU/iCAST graph
that was seeded back in v25.0 — the foundation finally has a visible
front door.

This is the first of three planned visualizations:
- **v25.4 (this release):** Lifecycle Diagram
- **v25.5 (next):** Threat Matrix Builder
- **v25.5.x (after that):** Role-Relationship Map

Splitting into three releases instead of one was a deliberate scope call —
each visualization is real work, and shipping incrementally lets you test
each one in production before the next builds on top.

## What's in v25.4

### The Frameworks page (`/frameworks`)

A new top-level page accessible from the nav. Three sections:

1. **Framework selector** — pills for AASE, CBEST, TIBER-EU, iCAST. AASE
   is selected by default and is the only one with full lifecycle data
   today. Selecting CBEST/TIBER/iCAST shows a "data coming in v25.6"
   placeholder (the framework headers are seeded but the phase/deliverable
   data isn't yet).

2. **Lifecycle Diagram** — horizontal four-node flow showing the AASE
   phases: Planning → Attack Preparation → Attack Execution → Closure.
   Equal-width nodes with a deliverable-count chip on each (Option C from
   the planning discussion — communicates content weight without distorting
   node sizes). Click any phase to drill into it.

3. **Phase Detail Panel** — sits below the diagram. When a phase is
   selected, shows:
   - Phase summary prose from the AASE PDF
   - Deliverables produced in this phase, each card showing the producing
     role and a teaser of the deliverable's contents
   - Key decisions an Exercise Director must answer in this phase

4. **Framework Concepts sidebar** — to the right of the detail panel.
   Lists the eight AASE concepts (Critical Function, Concession, Letter
   of Engagement, etc.) with "Universal" pills marking the ones that
   appear across all four frameworks. Click any concept to open the
   ConceptSidebar — same component the v25.1 scenario play loop uses,
   showing formal definition + cross-framework recognition pills.

### Top-nav entry

A new "Frameworks" entry sits between "Framework" (F3) and "Play". Both
"Framework" and "Frameworks" are in the nav. Naming option (i) from
planning — accept the collision now, consolidate in v26 if it proves
confusing.

### What's deliberately NOT in v25.4

- **No animated transitions between phases** — boring beats clever for a
  reference page. The diagram + panel just snap to selection.
- **No Sankey for phase flow** — Sankey visualizes flow with magnitude;
  AASE phases are a fixed sequence, not a quantitative flow. Wrong tool.
- **No force-directed layout for the lifecycle** — 4 nodes in a known
  sequence have no topology to discover. Force-directed would just jitter
  a chain. (Force-directed comes in v25.5.x for the Role-Relationship Map,
  which IS a network problem.)
- **No multi-framework comparison view** — that's v25.5 territory or later
  when CBEST/TIBER/iCAST have data to compare against.
- **No quizzes / no tutor / no scenario-creation** — this is a reference
  page, not a learning surface. Look-it-up, not test-yourself.

## Edge cases handled

- **Empty phases array** — defensive UI that says "Select a phase above"
  if for some reason no phase is auto-selected on load
- **OPTIONAL MATCH returning null deliverables** — the v25.0 server
  endpoint already filters these out; client also defensively
  `(phase.deliverables || []).length`
- **Long phase names** — truncate with ellipsis rather than wrap, so the
  diagram stays one row on every viewport
- **Narrow viewports** — diagram has horizontal scroll rather than wrap
  (preserving the visual sequence); detail panel and concepts sidebar
  reflow to single column when grid space drops below the breakpoint

## What's next — v25.5 plan

**Threat Matrix Builder.** A 5×5 grid (intent on Y-axis, capability on
X-axis, both: Very Low → Very High). Drag the 10 seeded threat actors
(Organised Cybercriminal Groups, Nation-State Tier 1-4, DDoS Extortionists,
etc.) onto the grid where you think they belong. Click "Reveal" — the app
shows where they actually are per AASE intelligence and explains the gap.

This is the genuinely novel interaction in the cyber-training space — none
of the competitors have a drag-and-drop threat-modelling exercise. About
1 week of work.

## Rollback

`git revert HEAD && git push` reverts everything. The /frameworks route
disappears, the nav entry disappears, the existing /framework page is
unaffected. Pure additive — nothing was changed in existing surfaces.
