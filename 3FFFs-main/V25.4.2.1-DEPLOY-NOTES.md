# v25.4.2.1 Deployment Notes — Hotfix: Stage field name (ISS-010)

**Caught by curl smoke test before user-facing impact** — the v25.4.2
"Practiced in" feature returned `stageTitle: null` for every stage
because the server endpoint queried `st.title` while the Stage node
schema uses `st.heading` for the display field.

This is the OBS-010 lesson from ISS-006 paying off — adding curl smoke
tests to the deploy notes caught a real bug at the API level before
any user opened the sidebar to see broken text.

## What was wrong

The v25.4.2 `practiced-in` endpoint:

```cypher
RETURN ...
       st.title AS stageTitle  ← Stage nodes don't have a `title` property
```

Stage nodes in the v25.0 / v25.1 / v25.3 seed data use `heading` as the
display field (matches the existing Scenario.jsx render code). My
endpoint authored in v25.4.2 made an incorrect assumption about the
schema.

The result was a payload like:

```json
{"stageId":"SC010-S1","stageOrder":1,"stageTitle":null}
```

If we'd shipped this to the UI, users would have seen "Stage 1 · " with
empty trailing space — looks broken. Curl caught it before that.

## What's fixed

**Server** — query corrected to use `coalesce(st.heading, st.title, 'Untitled stage')`:

- Tries `st.heading` first (correct for current schema)
- Falls back to `st.title` if a future seed uses that name
- Falls back to literal `'Untitled stage'` if both missing — defensive,
  shouldn't trigger in normal operation

**API contract** — payload field renamed from `stageTitle` to `stageHeading`
to honestly reflect the underlying schema. Easier to debug later when
someone wonders "why does the endpoint return stageTitle but the data
calls it heading?".

**Client** — ConceptSidebar updated to read `p.stageHeading` from the new
payload, with `|| 'Untitled stage'` fallback for safety.

## Lessons (carrying forward)

- **OBS-010 confirmed valuable.** Curl smoke test caught what the UI
  would have rendered as broken. Practice continues.
- **OBS-012 (new):** When authoring queries against existing graph
  schema, always verify field names against the seed data — don't
  assume by analogy. The Stage seed has `heading` because that's the
  word the original Scenario.jsx page used; my v25.4.2 query assumed
  `title` because that's a more conventional name. The graph honored
  what was seeded, not what I assumed.
