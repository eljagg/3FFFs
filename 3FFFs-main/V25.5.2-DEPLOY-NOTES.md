# v25.5.2 Deployment Notes — SC014 SOC defensive-tactical scenario (ISS-009 part 3 / final)

**The most operationally rich scenario in the catalog so far.** The
analyst plays a Tier 2 SOC analyst at Northgate Apparel during the
live CBEST test window (continues from SC013's setting). Six events
come across the SIEM in 48 hours; each stage shows one event and asks
the analyst to recognise the TTP, predict what's coming next, and pick
the right defensive response.

This release closes Sequence A. After v25.5.2 production confirmation,
Sequence A is fully deployed: per-framework concept summaries (v25.4.2),
Stage field hotfix (v25.4.2.1), MITRE foundation (v25.5), SC013
governance scenario (v25.5.1), and now SC014. Total: 5 releases over
the planned arc.

## Why this is the heaviest MITRE consumer in the catalog

Every stage in SC014 has a MITRE technique chip — eight chips total
across the eight stages (six primary + two consequence branches reuse
the parent stage's technique). Concept chips remain present (Critical
Function and Test/Halt Model are the two concepts in play) but the
MITRE chip is the primary "look up the playbook" affordance. This
proves the v25.5 foundation pays off operationally, not just
architecturally.

training is that the analyst sees the chain unfold one event at a
time, in real-world ambiguity (could it be the testers? could it be
real?), and has to make defensive decisions against a clock.

## OBS-013 substitutions in action

Two technique IDs from the original source documentation were not in
v18.1 Enterprise (T0040 is ICS, T1451 is Mobile). SC014 substitutes:

- **T1486 (Data Encrypted for Impact)** in Stage 6, replacing the
  source's T0040 reference for "compromise of critical function". T1486
  is the actual ransomware-on-ESXi mechanism observed in the 2025
  retail breaches. Better fit than the ICS substitute would have been.
- **T1566.004 (Phishing: Spearphishing Voice)** in Stage 1, replacing
  the source's T1451 Mobile SIM-swap reference. The 2025 retail TTPs
  led with vishing-the-helpdesk rather than SIM swap; T1566.004 is
  the correct Enterprise mapping.

## Source-material discipline

SC014 is authored from public threat reporting on the 2025 UK retail
intrusions plus the source documentation we already had. No proprietary
report content, no consulting-firm-specific naming. Where the public
reports use specific actor names ("Scattered Spider"), SC014 uses the
generic descriptor "an organised cybercrime group with retail-sector
specialisation" consistent with how the FCA and BoE describe these
actors in supervisory communications.

The 4-minute encryption rate, 256KB-segment pattern, AnyDesk on DCs,
and helpdesk-vishing-as-MFA-reset workflow are all directly from public
incident reports and threat-research write-ups. Nothing in SC014 leaks
non-public detail.

## What's new in the migration

The migration uses the **corrected idempotent-aware preservation
check pattern** — the SC013 migration false-positive on re-run is the
lesson driving this. Specifically:

```js
const expectedDeltas = sc014Existed
  ? { Scenario: 0, Stage: 0 }
  : { Scenario: 1, Stage: stagesAdded }
```

If SC014 doesn't exist yet, expect +1 Scenario and +8 Stage. If SC014
exists already (re-run), expect zero deltas in both. The check is
symmetric with the idempotent migration shape, so re-running the
migration after first-deploy doesn't produce a false-positive failure.

This is the pattern future scenario migrations will follow. v25.6 may
also retroactively fix the SC013 migration to match this pattern as
part of the freshness pass cleanup.

## Edge cases handled

- **Idempotent migration with symmetric preservation check.** Re-run
  produces zero deltas as expected; first run produces +1/+8 deltas as
  expected. False-positive elimination.
- **MITRE technique IDs verified against v18.1 seed.** All 6
  technique IDs referenced by SC014 (T1566.004, T1621, T1219,
  T1484.002, T1003.001, T1486) confirmed present in the v25.5 seed
  before authoring (OBS-012 lesson — verify field/data refs against
  existing seed before queries assume them).
- **Concept IDs verified against v25.0 seed.** Both concepts
  referenced by SC014 (CONCEPT-CRITICAL-FUNCTION and
  CONCEPT-TEST-HALT-MODEL) confirmed present.
- **No new helper functions or components.** SC014 ships with zero
  client-side code changes — pure data add. This is the smallest
  scenario release in Sequence A by file count.
