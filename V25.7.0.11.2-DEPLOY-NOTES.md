# v25.7.0.11.2 — Animation polish: control-stage feedback hint + Structuring (F1087) vertical compaction

Polish hotfix. Two issues from v25.7.0.11.1 testing addressed:

1. **Control toggles gave no feedback** when active at a stage where their signal wasn't live. Trainees toggled "Breach corpus monitoring," "Public bank-property minimization," and "Make customer LinkedIn private" at stage 7 of Gather Victim Information (F1067) and saw nothing happen. Visual confirms the control is active (checkmark + colored border) but the per-stage signal logic meant nothing surfaced. The control LOOKED dead.

## Pedagogical insight (OBS-027)

**Silent UI is broken UI.** A control that does something internally
but shows nothing externally trains the wrong reflex — trainees learn
"this control doesn't work" instead of "this control catches something
at a different stage." The fix preserves per-stage pedagogy (which is
what makes the controls genuinely teach) while making the control's
behavior legible.

## What changed and why

### Fix 1: Control-stage feedback hint (both engines)

**Before:** Toggle a control at a stage where its signal isn't live.
Control card shows checkmark + active border. Nothing else. Trainee
thinks the control is broken.

**After:** Toggle the same control. Card shows checkmark + active
border AND a new italic dashed-border hint underneath:

```
Active at stage 5 (Source 4), stage 7 (Handoff). Step through to view.
```

The trainee can now SEE that the control IS doing something, just
not at the stage they're currently viewing. The naming "Step through
to view" is meant to suggest the action they should take (use the
playback controls to step backward/forward).

Implementation:
- ControlToggle in both ProcessAnimation and TimelineThresholdAnimation
  now accepts `hasActiveSignalAtCurrentStage` and `stageLabels` props
- When `active && !naive && !hasActiveSignalAtCurrentStage` AND the
  control declares a `revealsAtStages` array, render the hint
- Hint pulls 1-2 word stage labels from each scene's stages array
  (so the hint shows e.g. "stage 5 (Source 4)" instead of bare "stage 5")
- All three existing animations already have `revealsAtStages` declared
  in their controls metadata — no scene-data changes needed

### Fix 2: Structuring (F1087) vertical compaction

**Before:** Each timeline panel was ~310px tall total (220px SVG +
60px header + 30px stats). Two panels = 620px. With queue + controls +
signals + section frame, total animation > 1200px tall. Required
scrolling on most screens.

**After:** Each panel is ~210px tall total (160px SVG + 24px
single-line header + 28px compact inline stats). Two panels = 420px.
Total animation ~900px — fits in a single 1080p viewport with room
for scrolling within the section if needed.

Three sub-fixes:

1. **SVG canvas: 220px → 160px.** Margins tightened (30/50 top/bottom
   → 22/38). Plot area drops 140px → 100px. Threshold line and
   transaction markers still read clearly.
2. **Single-line dense panel header.** Was a 2-row flex with name +
   descriptor on the left and account + age on the right. Now one row
   with bullet separators: `Mrs. Marcia Edwards · Mandeville market
   vendor · NCB ··6741 · 4yr customer`. Saves ~30px per panel.
3. **Compact inline stats strip.** Was a flex of 5 stat-items each
   stacked label-over-value. Now one inline row with label + value
   pairs and bullet separators: `DEPOSITS 23 · CUMULATIVE J$9.37M
   · AVG J$407K · ABOVE THRESHOLD 0`. "Declared occupation" dropped
   from the strip (already shown in the header descriptor — no need
   to repeat). Saves ~40px per panel.

Saves ~260px total across two panels. ProcessAnimation-driven
animations (IVR Discovery (F1073), Gather Victim Information (F1067))
untouched — they don't have this problem.

## Honest accounting

Three notes:

1. **The compact inline stats strip displays values that may wrap on
   narrow screens.** I chose flexbox with `flexWrap: 'wrap'` — on a
   typical desktop the four stats fit on one line; on a tablet or
   narrow viewport they'll wrap to a second line. That's acceptable
   degradation. If feedback says the wrapping looks awkward, I can
   add a media query later.

2. **"Declared occupation" was dropped from the stats strip** because
   it's already in the header descriptor. If you actually want it
   visible in BOTH places (some trainees may scan the stats and miss
   the header), I can add it back as a final stat-item. The current
   choice prioritizes vertical fit; tell me if you'd prefer redundancy.

3. **The control-stage hint renders on top of the existing "Would have
   flagged X of Y patterns" callout when both could apply** — but they
   can't both apply, because the catch-count callout shows for active
   non-naive controls regardless of stage, and the stage hint shows
   only when there's no active signal at the current stage. They're
   mutually exclusive in the actual UI. Just flagging that the layout
   handles both gracefully.
