# v25.7.0.4.1 — Two Views layout fix (full-width rendering)

**Hotfix on v25.7.0.4.** One file changed: `client/src/pages/Framework.jsx`.

## The bug

When v25.7.0.4 deployed and the migration ran, the Two Views visualization rendered correctly in terms of data (correct example, correct controls, correct interactivity). But it rendered visually broken — left column ~300px wide, right column ~210px wide, text wrapping and breaking mid-word.

## The cause

The Framework page renders all tactic visualizations in a fixed two-column layout: tactic text on the left (1fr), visualization panel on the right (1.05fr). That works for the kill-chain visualization (designed for that constraint) and the timeline visualization (designed for that constraint). It does not work for Two Views, which has its OWN two-column internal layout (cream banking dashboard + charcoal threat panel). Squeezing a 1100px-wide design into a 510px container splits each side to ~250px, neither of which fits the content.

I missed this in v25.7.0.4 because I tested the visualization in a standalone full-width prototype HTML file, then ported it to React assuming it would fit the same vertical real estate as the timeline. It does not — the timeline is a one-column scrubber design, Two Views is a two-column design wrapped in another two-column page layout. The squeeze compounds.

## The fix

Framework.jsx now classifies visualizations as "compact" (fit in the right-column viz panel as before) or "wide" (need the full body width). A `WIDE_VIZ_KINDS` set holds the wide kinds — currently just `'two_views'`. Compact visualizations render in the right column unchanged; wide visualizations render full-width below the two-column body. When a tactic has ONLY wide visualizations, the right column is dropped entirely and the text takes full width above the wide viz section.

This means on the FA0001 Positioning page:
- Tactic description (text) renders on the left
- PositioningTimeline renders compact on the right (its design fits there)
- PositioningTwoViews renders below, full body width (~1100px)

The Two Views internal `1.45fr / 1fr` grid then yields ~640px / 460px, which is enough room for both the banking dashboard and the threat panel + 2x2 controls grid.

## OBS-029 (new): test visualizations IN their host page, not just standalone

I built and tested the prototype as a full-width standalone HTML page, then ported to React without verifying how it'd render INSIDE the Framework page's right viz column. This is the second time a visualization shipped with a layout regression that was visible only in the host context (the first was OBS-028 — the v25.7.0.3 timeline rendering issue, also a host-context bug). Logging this:

> Before declaring a visualization complete, render it inside its actual host page at the actual host viewport width, not just standalone in a 1180px max-width prototype. Visualizations that contain their own multi-column or fixed-aspect layouts especially need this check, because the host page may give them less horizontal real estate than the prototype assumes.
