## Pedagogical insight (OBS-027)

**Comprehension is a function of attention placement.** The existing
`caption` field on every stage of every animation is purpose-built
to explain what's happening at that stage — but it lives in a
static area below the animation, requiring the trainee to look
*away* from the visual to read the explanation, then look back.
That eye-movement during a 6-9 second stage means the trainee
either misses the visual or misses the caption. By surfacing the
caption as a semi-transparent overlay at the bottom of the
animation canvas, the engine puts the explanation in the trainee's
existing visual focal area — they read while they watch.

The toggle exists because *some* trainees benefit from unaided-
mode passes (they want to test their own comprehension before
reading the explanation). Default is ON because the primary use
case is teaching, not testing. Mirrors lecture-capture closed-
caption defaults.

## Design decisions (locked v25.7.0.17)

- **Position:** bottom of the animation canvas, semi-transparent
  dark background (slate-900 at 92% opacity) with light slate-100
  text. Slides up from below on stage change (0.35s ease-out).
  `pointer-events: none` so it never blocks clicks on the diagram.
- **Default state:** ON for first-time visitors. Trainees explicitly
  opt out if they want unaided mode.
- **Persistence:** per-session via `localStorage` key
  `'3ffs-coach-on'`. Trainee picks their preferred mode once;
  preference carries across all animations and survives page
  refresh. Defensive against private-browsing modes where
  localStorage throws — falls back to in-memory state.
- **Source:** reads `stage.caption` directly. No new content
  fields authored — the captions already cover the pedagogy.
- **Toggle button:** 💬 emoji in the playback bar, next to the
  audio mute toggle. Opacity + tooltip communicate state. Tooltip
  text differs based on state ("Coach mode on — click to hide
  stage captions" / "Coach mode off — click to show").

## What ships in this release

### NEW: shared component file

`client/src/components/framework/animations/StageCoachOverlay.jsx` —
~120 lines. Three exports:
- `useCoachToggle()` — React hook returning `[isCoachOn, toggleCoach]`
  with localStorage persistence
- `<StageCoachOverlay isOn text stageLabel stageIdx />` — the
  overlay component itself
- `<CoachToggleButton isOn onToggle baseStyle />` — the button
  component for engine playback bars

The component is engine-agnostic — same code used by all three
engines. Future engines just import these three exports.

### Three engines wired

`ProcessAnimation.jsx`, `MultiActorSequenceAnimation.jsx`, and
`TimelineThresholdAnimation.jsx` each:
- Import the three exports from StageCoachOverlay
- Add `const [isCoachOn, toggleCoach] = useCoachToggle()` next to
  the existing `isMuted` state
- Render `<StageCoachOverlay>` inside a relative-positioned wrapper
  around the canvas area
- Render `<CoachToggleButton>` in the playback bar's right side,
  before the audio toggle

ProcessAnimation also gets `position: relative` added to its
`canvas` style. MAS and Timeline wrap their existing canvas
content in a new `<div style={{ position: 'relative' }}>` (the
existing styles already worked without this, but the overlay needs
absolute positioning relative to the canvas, not the page).

## Honest accounting

Three notes:

1. **Captions may feel long for some stages.** I deliberately
   chose Interpretation A (surface existing captions) over
   Interpretation B (author new short coach lines) to ship in
   one turn rather than across multiple turns of authoring +
   iteration. If after live-trainee testing the captions feel
   too dense for the live-overlay use case, the right fix is to
   revise the captions themselves — they should serve both the
   static-reading and live-overlay use cases. Don't add a second
   content field unless the dual-use approach genuinely fails.

2. **localStorage was forbidden in artifacts but allowed in
   production.** This was the third decision you said yes to —
   the production codebase has no constraint, and the value of a
   sticky preference (trainee picks their mode once) significantly
   outweighs the cost of a 3-line try/catch. The hook is defensive
   against private-browsing edge cases.

3. **No scene files changed.** The overlay reads existing caption
   text directly. Future scene authors don't need to do anything
   different — every stage they write a caption for automatically
   gets a working coach overlay.
