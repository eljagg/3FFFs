## Four-lens block (OBS-018)

Same lens applies as v25.7.0.9 — this hotfix doesn't change pedagogical
content, only the frame and section-level affordances. All four lenses
benefit:

- **Teller / Analyst / SOC / Executive:** Once the description has been
  read, it becomes visual noise. Collapsing it gives the trainee more
  vertical real estate for the animation, controls, and cross-references
  — the parts they're actively engaging with. Different roles will
  collapse different sections at different times depending on their
  focus. The pattern generalizes to future content-heavy sections.

## Pedagogical insight (OBS-027)

**Information that has served its purpose should be retractable.** The
Description tells the trainee what the technique IS — once they know,
keeping it permanently visible competes for attention with the active
content (the animation, the controls, the cross-references they're
exploring). The same applies to the Animation section once they've
watched it; they may want to study the cross-references next without
the animation's vertical real estate dominating.

The lever this enables: training the trainee's reflex that learning
isn't passive consumption of fixed content — it's an iterative process
of focusing on different aspects at different times. The UI should
support that, not enforce a single fixed reading order.

## The two changes

### Change 1 — Slightly brighter focal-zone signaling

Per Omar's feedback that v25.7.0.9.1's signaling "looks very good, can
be a little brighter still":

**Was (v25.7.0.9.1):**
- Scale: 1.015×
- Outer glow opacity: 40 (~25%)
- Inner accent tint opacity: 33 (~20%)

**Now (v25.7.0.9.2):**
- Scale: **1.020×** (small bump)
- Outer glow opacity: **60 (~38%)** — slightly more presence
- Outer glow radius: **28px** (was 24px) — softer edge
- Inner accent tint opacity: **40 (~25%)** — slight bump
- Border (3px), pulse rate (1.6s), ACTIVE label: unchanged

The cumulative effect is more presence without crossing into
"distracting." The pulse remains low-intensity. ACTIVE label still
provides verbal cue alongside visual.

### Change 2 — Collapsible Description and "How this technique works" sections

Per Omar's feedback that "once we finished reading, the description is
no longer necessary visible":

**Behavior:**
- Description and "How this technique works" section headers become
  click-targets (chevron rotates to indicate state)
- Click header → section body collapses (stays in DOM as 0-height when
  collapsed); click again → re-expands
- **Default state: expanded** — first-time users need to read the
  description and see the animation
- **Collapse state resets on technique change** — both outer prop
  changes (clicking a different technique in the grid) AND internal
  navigation (clicking sibling/parent/child link in sidebar). Each new
  technique starts fresh.
- **Animation auto-pauses on collapse** — via an `externalPauseSignal`
  counter prop on ProcessAnimation. When the animation section
  collapses, the counter bumps; ProcessAnimation watches the prop and
  pauses when it changes. Animation retains its current stage; user
  clicks Play to resume after re-expanding. Animation does NOT
  auto-resume on expand — user attention should be the trigger, not
  the section state.

**Other sections unchanged** — Roles, Mitigation, Cross-references,
Sub-techniques, and Siblings remain always-visible. They're short
enough that collapsibility doesn't help much. (If feedback later says
"I want all sections collapsible" it's a small follow-up — the
Section component already supports both modes via the optional
`collapsible` prop.)

**Visual treatment of collapsible headers:**
- Same typography as non-collapsible headers (uppercase mono caps,
  letter-spacing, faint color)
- On hover: header text shifts to accent color (small affordance
  that says "this is interactive")
- Chevron `→` rotates 90° when expanded
- aria-expanded attribute set correctly for screen readers

## Honest accounting

Three notes:

1. **The signaling bump is small** — 1.015 → 1.020 scale, 40 → 60 glow
   opacity. Cumulative effect should feel "brighter" without being
   aggressive. If on first viewing it still feels too subtle, easy
   single-character tweak — change 60 to 75, 1.020 to 1.025. If it
   feels overdone, drop the inner tint or reduce the glow radius.
   Backout cost: ~3 lines.

2. **Collapse state isn't persisted across sessions.** Close the
   sidebar, reopen, you get expanded defaults again. Per-session
   memory wasn't requested and adding it (sessionStorage) means more
   surface area to test. Easy to add later if useful.

3. **Animation pause-on-collapse uses a counter, not a boolean.**
   Reason: a boolean toggle ("paused: true") that flips back to
   "false" when expanded would either auto-resume the animation
   (intrusive) or do nothing (in which case why have a boolean).
   Counter-bump pattern is clean: every collapse increments, the
   animation pauses, expand does nothing extra (animation stays at
   its current stage, user controls resume). Simpler mental model.

This release is the kind of polish that compounds. Every future
animation we ship has this collapse pattern available. Future
techniques with long descriptions get the benefit. The ProcessAnimation
engine now also has a clean external-pause hook that future host
contexts can use.
