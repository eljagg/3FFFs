# v25.7.0.4.6 — Wider page on F3 Framework

You wanted the cream dashboard to actually use the available horizontal space, not be constrained to the page's default 1100px reading width. I was anchoring to the OM avatar but treating that as the page-content edge — when actually you wanted the dashboard to extend WIDER than the rest of the page body.

## What changed

`Page.jsx` now accepts a `wide` prop. When set, max-width bumps from 1100 to 1500 — for pages that have wide content like data tables, side-by-side visualizations, or full dashboards.

Framework.jsx passes `wide` to its Page wrapper. The whole F3 Framework page now uses 1500px max-width — wider tactic list, wider lede paragraph, and most importantly, the cream dashboard renders at the full 1500px page-content-width.

Other pages (Home, Scenarios, etc.) keep the 1100px default — they're editorial/marketing content where reading line-length matters more.

## Files changed

- `client/src/components/Page.jsx` — adds optional `wide` prop
- `client/src/pages/Framework.jsx` — passes `wide` to Page

`PositioningTwoViews.jsx` is unchanged from .5.
