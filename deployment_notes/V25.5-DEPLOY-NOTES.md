# v25.5 Deployment Notes — MITRE ATT&CK foundation (ISS-009 part 1)

**Establishes the MITRE ATT&CK Enterprise schema in the graph.** 128
techniques across 12 attack-phase tactics, all 14 tactics seeded as
nodes, plus the new sidebar component that scenario stages will use
in v25.5.1+ to surface technique detail.

This is the **foundation only**. No scenarios in v25.5. SC013 (CBEST
governance, parallels SC010) layers in v25.5.1; SC014 ("Inside the
Service Desk Compromise", defensive-tactical SOC-tier) layers in v25.5.2.

## What's seeded

### 14 MitreTactic nodes

All Enterprise tactics including the two pre-attack tactics
(Reconnaissance, Resource Development) for completeness, even though
no techniques in this seed map to them. Properties: `id` (TA####),
`shortName`, `name`, `order` (1-14), `description`, `phase`
(`pre-attack` or `attack`), `vintage`.

### 128 MitreTechnique nodes

Filtered from the 691-technique full ATT&CK Enterprise corpus to those
relevant to banking/retail attack surface. Selection criteria:

- All technique IDs explicitly referenced in the v25.5+ source
  documentation for SC014 (the defensive-tactical scenario)
- Parent techniques of every selected sub-technique (so users get
  context when they look up a sub-technique)
- Top-level techniques widely observed in financial-sector incidents
  per FS-ISAC, CISA financial-sector advisories, and major threat
  reports
- High-frequency sub-techniques where the parent alone would lose
  operationally relevant detail (e.g. T1110.003 Password Spraying)

### Edges

- **OF_TACTIC**: technique → tactic. Some techniques map to multiple
  tactics (e.g. T1078 Valid Accounts spans Initial Access, Persistence,
  Privilege Escalation, Defense Evasion). Around 145 edges total.
- **SUB_TECHNIQUE_OF**: sub-technique → parent technique. 61 edges.

No `:USES_MITRE_TECHNIQUE` edges from `:Stage` nodes are created in
v25.5 — that happens in v25.5.1 when SC013 ships.

the Spider", May 2025) are not in v18.1 Enterprise and were excluded
from the seed:

| **T0040** | ICS ATT&CK (industrial control systems matrix), not Enterprise. The source doc cited it for "compromise of critical function" generically. |
| **T1451** | Mobile ATT&CK ("SIM Card Swap"), not Enterprise. The source doc cited it for the SIM-swap precursor to the helpdesk vishing. |

When SC014 is authored in v25.5.2, the SIM-swap concept will be mapped
to the closest Enterprise equivalent — likely **T1078 (Valid Accounts)**

depending on the specific scenario stage. This is a v25.5.2 authoring
decision; v25.5 just notes it.

The other 27 source-document IDs are all in this seed.

## Edge cases handled

- **Preservation enforced.** Migration aborts if any existing node or
  relationship count changes. Backstop against accidental destructive
  edits as the seed grows.
- **Idempotent.** Re-running the migration sets the same properties
  on the same nodes. Safe.
- **Missing tactic warning.** If a technique seed references a tactic
  not in the tactics seed (shouldn't happen but defensive), the
  migration logs a warning and continues without creating that
  OF_TACTIC edge.
- **Cypher post-aggregation scope strict** (OBS-011 lesson). All
  queries in `mitre.js` use either no aggregation, or aggregation in a
  single WITH that doesn't try to reference variables past the
  projection.
- **Stage field name uses heading not title** (OBS-012 lesson). The
  practiced-in query mirrors the v25.4.2.1 pattern with
  `coalesce(st.heading, st.title, 'Untitled stage')`.

## Rollback

`git revert HEAD && git push` reverts code. Graph cleanup if needed:

```cypher
// Remove all MITRE nodes and their edges
MATCH (n) WHERE n:MitreTactic OR n:MitreTechnique
DETACH DELETE n;

// Drop constraints
DROP CONSTRAINT mitre_tactic_id IF EXISTS;
DROP CONSTRAINT mitre_technique_id IF EXISTS;
```

Both blocks idempotent. Existing SC001-SC012 unaffected.

## Lessons captured

- **OBS-013 (new):** ATT&CK contains parallel matrices (Enterprise,
  Mobile, ICS) with overlapping technique-ID ranges. T0040 is ICS;
  T1451 is Mobile; T1xxx is Enterprise. When ingesting third-party
  threat reports that cite technique IDs, verify the matrix before
  assuming Enterprise. The CovertSwarm document's mis-citation of
  T0040 and T1451 caught this for us.
