# v25.7.0.4.3 — Working full-bleed + compact techniques grid

**Hotfix on v25.7.0.4.2.** Fixes the broken full-bleed (which was clipping the cream dashboard) and compacts the techniques list so the page no longer requires endless scrolling.

## What went wrong in v25.7.0.4.2

I attempted full-bleed using `calc(50% - 50vw)` from inside `tacticBody`. The math doesn't escape `tacticBody`'s nested padding (`paddingLeft: 84, paddingRight: 40`), so the element computed an off-center bleed and got CLIPPED by `overflowX: hidden` that I'd added to `tacticBody` to suppress horizontal scrollbars. End result: cream dashboard rendered too far right, content cut off on the left.

I created the bug while trying to prevent a different problem.

## What this release fixes

**Three changes:**

### 1. Full-bleed actually works now

The wide-viz section is now rendered as a SIBLING of `tacticBody`, not a child. From inside the Page wrapper (which is the parent in the new structure), the `calc(50% - 50vw)` math only has to escape ONE level of centering — the Page's max-width:1100 + margin:auto. No nested padding chain to fight.

The `overflowX: hidden` containment moved from `tacticBody` (where it was clipping the bleed) to a new outer wrapper inside `Page.jsx`. This stops the 100vw width from triggering horizontal scrollbars without clipping the bleed itself.

### 2. Compact techniques grid

The techniques list was rendering 25 cards each with full descriptions clamped to 4 lines, in a `minmax(280px, 1fr)` grid — that's roughly 13 rows of techniques, requiring substantial scrolling.

New: compact dense grid `minmax(190px, 1fr)`, only ID and name shown per card (no description), tighter padding. Same 25 techniques fit in ~5 rows. Description is still available on hover (via the `title` attribute on each card).

### 3. Layout structure refactor

The expanded-tactic body is now structured as three sibling sections inside the tactic wrapper:

```jsx
<>
  <div style={tacticBody}>            {/* lede + compact viz */}
    {textBodyLede + compact viz}
  </div>
  <div style={tacticBodyWideViz}>     {/* full-bleed wide viz */}
    {wide viz}
  </div>
  <div style={tacticBody}>            {/* techniques */}
    {techniques}
  </div>
</>
```

This makes each section's bleed/padding independent. Wide viz can be edge-to-edge; the other sections respect normal page margins.

## Honest reflection

This is the third layout iteration on a single visualization. Each step revealed something I hadn't predicted:

- v25.7.0.4: didn't anticipate the host page's two-column constraint
- v25.7.0.4.1: shipped a fix that used the right-column properly but forgot order matters (cream dashboard ended up below the techniques list)
- v25.7.0.4.2: added the bleed but used the wrong CSS structure (nested inside padded container) which clipped instead of bled
- v25.7.0.4.3: this one — proper sibling structure for the bleed + compact techniques

Logging another observation:

> **OBS-030:** When implementing full-bleed elements with calc(50% - 50vw), render the bleed element as a sibling of the page wrapper's CONTENT box, not nested inside containers with their own padding. Each level of nested padding throws off the math by exactly that padding amount, and the overflow-x containment that prevents scrollbars also clips bleed elements if applied to the wrong ancestor.
