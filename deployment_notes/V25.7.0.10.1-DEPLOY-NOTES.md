## Four-lens block (OBS-018)

This release affects readability of project documentation, not user
experience. Lens-by-lens:

- **Teller / Analyst / SOC / Executive:** None of these roles read
  ANIMATION-TRIAGE.md directly — it's a project-internal methodology
  record. The "lens" beneficiaries here are FUTURE Claude sessions
  (next session, fresh context, no conversation history) who will read
  this file as context. Without paired names, future-Claude has to
  cross-reference each F-code against the live framework to know what
  it means; with paired names, future-Claude can read the file as
  self-contained context.

## Pedagogical insight (OBS-027 one-pedagogical-insight-per-release)

**Documentation that requires lookup is documentation that doesn't get
read.** A reference file full of bare codes (F1097, F1067, F1073)
forces the reader to maintain mental state ("which one was 1097
again?"). A reference file with paired names (F1097 3DS Bypass)
collapses that lookup into the document itself. The cost is small
(~50 extra characters per code mention); the readability gain
compounds with every reading.

The lever this release enables: a project documentation surface that
stays self-contained and readable as the framework grows beyond what
any single person can hold in head at once.

## Honest accounting

Two notes:

1. **The "F-code TBD" entries** for sub-threshold structuring, mule
   pipeline cross-bank funds flow, card-not-present fraud, and wire
   transfer fraud are deliberate. I considered querying the live
   framework API to resolve those F-codes now, and chose not to.
   Reason: the F3 framework F-codes for those concepts may evolve as
   the framework is authored. Recording a code that might change
   creates documentation debt. Better to resolve when we actually
   start each animation.

2. **The dual-name convention for F1008.001** ("Change e-delivery
   settings (the silent alarm move)") — official name first,
   pedagogical descriptor second — is a pattern I'd like to use for
   any technique where the official name is dry but a pedagogical
   descriptor better captures the lesson. F1008.001 is the canonical
   example. Future ANIMATE candidates might also benefit from this
   pattern; we'll see when we author them.

This is the smallest release in the v25.7.0.x series so far —
intentionally. Documentation cleanup should be cheap and frequent,
not bundled into bigger code releases where it gets deferred.
