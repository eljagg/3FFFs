# v24.3 Deployment Notes — page-vs-card layering for a polished feel

**What this fixes:** the whole app was reading as one flat tone — page
background, content cards, signal panels — all visually merged into a single
field of cream (or single field of dark). No hierarchy, nothing "popped".

## The fix

Two changes work together:

**1. CSS — page background darkens, card surface stays bright.**

Page bg (`--paper`) shifts to a noticeably darker tone. Card surface
(`--paper-hi`) stays where it is. The luminance gap between them widens
substantially, which is what makes cards read as elevated sheets sitting
on a page rather than disappearing into it.

| Token | Light v24.2 | Light v24.3 | Dark v24.2 | Dark v24.3 |
|-------|-------------|-------------|------------|------------|
| `--paper` (page bg)        | L=88.0 | **L=76.7** | L=0.65 | **L=0.48** |
| `--paper-dim` (inner panel)| L=79.2 | L=83.0     | L=1.09 | L=1.09     |
| `--paper-hi` (card)        | L=93.0 | L=94.6     | L=1.22 | **L=1.57** |
| **page → card delta**      | 5.0pp  | **17.9pp** | 0.57pp | **1.09pp** |

Light mode page-to-card spread more than triples. Dark mode doubles. Both
make the existing `--paper-hi` cards read as elevated without any per-page
code changes.

**2. Scenario page — stage panel and completion panel get a polished elevation.**

The two main cards on the Scenario page (the stage panel and the completion
panel) get:
- Larger radius (14px instead of 10px)
- Theme-aware drop shadow (subtle in light mode, deeper in dark)
- Slightly more generous padding (32/36 instead of 28/32)

The shadow uses two new shared CSS tokens (`--shadow-card`, `--shadow-card-dark`)
defined in `:root` so future pages can adopt the same elevation by adding
`className="elevated-card"` — the new utility class for opt-in shadow treatment.

## Pages affected

**Direct visual change (every page):** because `--paper` is now darker, every
existing `var(--paper-hi)` card across the app — Home stat tiles and action
cards, Coverage heatmap, Tutor messages, Framework tactic accordion, Matrix
cells, AdminUsers tables, Quiz options — automatically reads as elevated.
Zero code change in those pages. They look better for free.

**Targeted polish (Scenario page only):** stage panel and completion panel
get the larger radius + drop shadow treatment as showcase pages. The app's
feature pages (Home, Coverage, Tutor, Framework, etc.) keep their current
look but with better page contrast — they can be upgraded to the
`.elevated-card` treatment one at a time as we revisit each.

## What to test

1. **Light mode, open a scenario.** The stage panel should now read as a clear
   white card sitting on a sandier, slightly tan page background. The drop
   shadow should be visible but subtle — not heavy.
2. **Dark mode, open the same scenario.** The card should sit on a deeper
   page bg with a soft shadow underneath. Body text is unchanged in
   contrast (still WCAG AAA against the new card).
3. **Walk other pages** — Home, Coverage, Quiz, Tutor, Framework, Matrix.
   In every case the existing `var(--paper-hi)` cards should now read as
   distinct from the page bg. No code changes there but they should all
   look better.
4. **Switch theme via the toggle.** Both modes should look intentional.

## Rollback

`git revert HEAD && git push` — Railway redeploys v24.2.

## What comes next

Once this lands and the look is right, the `.elevated-card` utility class
makes it a 1-line-per-page change to bring the matching shadow/radius
treatment to Home, Coverage, Tutor, etc. We can do those page-by-page when
we revisit each, or as a batch in a single later release.
