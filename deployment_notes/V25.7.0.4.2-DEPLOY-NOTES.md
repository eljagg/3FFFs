# v25.7.0.4.2 — Reorder + full-bleed for Two Views

**Hotfix on v25.7.0.4.1.** Layout redesign of the expanded tactic section so the cream Two Views renders BETWEEN the description and the techniques list, AND spans the full viewport width edge-to-edge.

## Why this exists

After v25.7.0.4.1 deployed, the cream banking dashboard rendered correctly but at the WRONG position on the page — all the way at the bottom, below the entire techniques list. Users had to scroll past 25+ technique cards to reach it. By the time they arrived, the context from the tactic description was lost.

Additionally the Two Views, despite being a "wide" visualization, was constrained to the page's 1100px max-width. The user wanted it edge-to-edge across the actual viewport.

## What ships

Two files modified:

**client/src/pages/Framework.jsx:**
- Splits `textBody` into `textBodyLede` (description + takeaway) and `techniquesBlock` (the techniques list)
- Restructures the expanded-tactic body into three ordered sections:
  1. Two-column section: lede (left) + compact viz like the timeline (right)
  2. Wide visualizations — full bleed, edge-to-edge viewport width
  3. Techniques list — full body width, below the wide viz
- Adds `overflowX: 'hidden'` to `tacticBody` to contain the 100vw wide-viz section without triggering horizontal scrollbars
- Adds styles for the full-bleed container, the inner card, the title-aligned header, and post-viz techniques spacing

**client/src/components/visualizations/PositioningTwoViews.jsx:**
- Removes outer border and border-radius from the stage wrapper
- Adds top + bottom hairlines instead — to mark the visualization's boundary against surrounding page content without creating rounded corners floating at viewport edges

## How the full-bleed works

Standard CSS technique:

```js
marginLeft:  'calc(50% - 50vw)',
marginRight: 'calc(50% - 50vw)',
width:       '100vw',
```

The element gets pulled left/right by (viewport-half minus container-half) regardless of how deeply nested it is in margin/padding chains. The bleed container is `overflow:hidden` on its parent so the 100vw doesn't trigger horizontal scrollbars on browsers where viewport math includes the scrollbar.

## Section order — before vs after

Before:


After:
