# v25.4.2 Deployment Notes — Concept enrichment (ISS-008)

**Resolves the ConceptSidebar's lack of per-framework explanations.** The
"Recognised across all four frameworks" pill row from v25.1 is now an
expandable card grid where each framework gets its own summary content.

Per Sequence A1 (approved): AASE-side summaries are populated from the
existing v25.1 concept content. CBEST, TIBER-EU, and iCAST cards render
a "v25.6 — content pending" placeholder until the freshness pass fills
them in against current framework documentation.

Bonus shipped: **"Practiced in" section** on every concept, listing the
scenarios + stages where it's tested. Click any to jump straight into
that stage.

This is **v25.4.2 in Sequence A** — the smallest, lowest-risk piece of
the four-release plan. v25.5 (MITRE foundation) follows.

## What changes for the user

### 1. Per-framework cards in the ConceptSidebar

Open SC010 → click "Look up: Critical Function". Where you previously saw
a row of four pills (`AASE · Singapore`, `CBEST · United Kingdom`, etc.),
you now see four expandable cards:

```
┌─ AASE · SINGAPORE                                v2018  ▼ ─┐
│  A business function or service that, if compromised…    │
│  Sourced from AASE 2018 reference.                       │
└──────────────────────────────────────────────────────────┘
┌─ CBEST · UNITED KINGDOM           [pending]           ▼ ─┐
│  (collapsed by default — click to see pending notice)    │
└──────────────────────────────────────────────────────────┘
┌─ iCAST · HONG KONG                [pending]           ▼ ─┐
└──────────────────────────────────────────────────────────┘
┌─ TIBER-EU · EUROPEAN UNION        [pending]           ▼ ─┐
└──────────────────────────────────────────────────────────┘
```

AASE is sorted to the top and expanded by default (it has content). The
others are collapsed but still visible — opening one reveals the
"content pending" placeholder, so users understand the three other
frameworks recognise the concept but the framework-specific gloss
hasn't been authored yet.

### 2. "Practiced in" section

Below the framework cards, a new section lists every scenario stage
that tests this concept. Example for "Critical Function":

```
PRACTICED IN 4 STAGES

[SC010] Stage 1 · Scoping the Exercise              ↗
        Commissioning Your First AASE — First Carib…

[SC010] Stage 2 · Selecting Critical Functions      ↗
        Commissioning Your First AASE — First Carib…

[SC011] Stage 3 · Mid-exercise critical-function r… ↗
        Risk Management Mid-Exercise

[SC012] Stage 4 · Post-exercise control validation  ↗
        Reading the Defence Report
```

Click any entry to close the sidebar and jump directly into that stage.
Closes the loop between reference content (look up the definition) and
applied learning (here's where you actually use it).

### 3. Examples reordered above frameworks

Examples are concept-level (apply across all frameworks), so they now
sit above the per-framework cards. Reading flow becomes: **What is this
concept? → Examples of it. → How does each framework treat it? → Where
do I practice it?**

## Edge cases handled

- **Migration idempotent.** Rerunning sets the same edge summaries
  again. Verified by the preservation check (edge count unchanged).
- **Edge summary fallback.** If for any reason the migration hasn't
  run, the route falls back to `Concept.summary` on the node — the
  UI still renders, just with the same content for all frameworks.
- **Empty practicedIn.** Concepts with no `:TESTS_CONCEPT` edges
  (none currently, but defensive) just don't render the Practiced In
  section.
- **Stage order as Neo4j integer.** The endpoint converts `Integer`
  objects to plain JS numbers so the URL param is correct.

## Rollback

`git revert HEAD && git push` reverts both client and server changes.
The migration's edge properties stay (won't be read by reverted code,
just orphaned data). To clean those:

```cypher
MATCH (k:Concept)-[r:APPEARS_IN_FRAMEWORK]->(f:Framework)
REMOVE r.summary, r.pending, r.vintage, r.updatedAt;
```

Safe and idempotent.
