## Four-lens block (OBS-018)

Same lens applies as v25.7.0.9 — this hotfix doesn't change pedagogical
content, only the frame in which it's presented. All four lenses
benefit from reduced scrolling and clearer focal signaling:

- **Teller / Analyst / SOC / Executive:** All four were fighting the
  scrollbar to reach detection controls. Wider sidebar + tighter
  vertical layout means controls + canvas + caption are simultaneously
  visible on a typical 1080p+ display. The brighter focal signaling
  also helps users whose eyes don't catch subtle motion changes — a
  population that includes anyone reading on a smaller laptop screen,
  anyone with non-perfect color vision, anyone using browser zoom.

## Pedagogical insight (OBS-027)

**A pedagogical animation that requires scrolling to use is teaching
less than the same animation that fits on screen.** The cognitive cost
of "scroll, find the control, scroll back to see the result" is
non-trivial — it disrupts the trainee's focus on the content. The
v25.7.0.9 sidebar at 820px was a conservative first call ("it's a
sidebar, not a modal") that didn't reflect the actual interaction
pattern (focused content + dimmed backdrop = effectively a modal).
This hotfix corrects that without changing any of the animation's
substantive content. **The lever this enables: reading the testing
feedback honestly and prioritizing 1-character-of-actual-improvement
over preserving the original design's elegance.**

## The three changes

### Change 1 — Wider sidebar for animation-equipped techniques

**Was:**
```javascript
maxWidth: technique && ANIMATION_MAP[technique.id] ? 820 : 600
```

**Now:**
```javascript
maxWidth: technique && ANIMATION_MAP[technique.id]
  ? 'min(1400px, 95vw)'
  : 600
```

Behavior:
- 1920px monitor → 1400px sidebar (capped for readability)
- 1440px laptop → 1368px sidebar (95% of viewport)
- 1280px laptop → 1216px sidebar (95% of viewport)
- 1024px (rare) → 972px sidebar (95% of viewport)

Non-animation techniques **stay at 600px**. Per Q-3 confirmed: text
content reads better in narrower columns (typographic best practice
60-80 characters per line). Wider sidebars apply only when the
animation needs the room.

### Change 2 — Brighter focal-zone signaling

**Was:** 1.005× scale + 2px border + faint shadow. Subtle to the point
that some users couldn't tell which zone was active.

**Now:** 1.015× scale + 3px border + outer glow + inner accent tint +
explicit "● ACTIVE" indicator pulsing in the focal zone's title strip.

The pulse is intentionally low-intensity per Q-2 (a heartbeat at 1.6s
period, opacity 0.5 → 1.0 → 0.5, easeInOut) — visible without being
distracting. No strobe, no jarring animation.

The verbal label "ACTIVE" lands for users whose eyes don't catch
subtle motion changes — accessibility win that pays back beyond the
specific feedback.

### Change 3 — Layout reflow for wider canvas

Three coordinated grid adjustments to take advantage of the new
horizontal real estate:

- **Three-zone canvas grid:** `minmax(180px, 0.85fr) minmax(220px, 1.15fr) minmax(180px, 0.85fr)` → `minmax(220px, 1fr) minmax(280px, 1.3fr) minmax(220px, 1fr)`. All zones gain breathing room; middle zone keeps its slight prominence.
- **Canvas minimum height:** 460px → 420px. Zones are now wider, so content fits in less vertical space — overall canvas is shorter, which is exactly what frees up vertical room for controls without scrolling.
- **Detection controls grid:** `repeat(auto-fill, minmax(240px, 1fr))` → `repeat(auto-fill, minmax(280px, 1fr))`. Wider sidebar means each control card can be a bit bigger; meta text (the "skipping voice prompts <0.5s consistently" descriptions) reads cleaner.

Net effect: animation comfortably fits on a typical 1080p+ display
without scrolling, controls are easier to scan, focal-zone changes
are unmistakable.

## Honest accounting

This release is small and focused. Three honest notes:

1. **The min(1400px, 95vw) ceiling** — I picked 1400 based on
   typographic readability (you don't want lines of text that are
   1800px wide on an ultra-wide monitor). On 4K+ monitors, the
   sidebar will look "small" relative to the screen, but the content
   is correctly sized for reading. If feedback says "I want more on
   4K," we can revisit — possibly with a max more like 1600 or 1800.

2. **The ACTIVE pulse uses a 1.6s heartbeat period.** Slow enough not
   to distract (which would be ~0.5-0.8s territory), fast enough to
   be perceptibly alive. If it feels too slow or too fast in
   practice, easy single-line tweak.

3. **The brighter focal signaling stacks 4 effects** (3px border + outer glow
   + inner tint + 1.015 scale). That's deliberate — each effect alone
   is too subtle (the original had just 2 of them); combined they
   become unmistakable without any single one being aggressive. If
   the result feels overdone, easiest fix is to drop the inner tint
   first (it's the most "decorative" of the four).

This hotfix is the kind of release that pays for itself many times
over — every future animation we ship inherits the better frame. The
~1 hour of work here saves rework on v25.7.0.10's OSINT animation,
v25.7.0.11's different-shape animation, and every animation after
that.
