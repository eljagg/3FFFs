# v25.1 Deployment Notes — SC010 scenario + ConceptSidebar (Phase 2 of 5)

**This release adds the first analyst-tier scenario to the platform.** SC010,
"Commissioning Your First AASE — First Caribbean Trust", walks the learner
through the seven planning-phase decisions a security manager must make
when commissioning a red-team exercise at a Caribbean bank: critical-function
scoping, secrecy level, working-group composition, provider selection,
concession policy, escalation-path interception, and Letter-of-Engagement
contents.

It also introduces the **ConceptSidebar** — a slide-over component that
fetches AASE Concept definitions live from the graph (`/api/frameworks/concepts/:id`)
when the analyst clicks the "Look up: <Concept>" affordance on a stage.
This is the visible proof that the v25.0 graph is doing real work.

Prerequisites: **v25.0 must already be deployed and migrated.** The new SC010
stages link to Concept and FrameworkPhase nodes seeded by `migrate:frameworks`.
The v25.1 migration script checks for v25.0 presence and aborts with a clear
error if it's missing.

## Rollback

`git revert HEAD && git push` reverts the code. The graph nodes added by
the migration stay in Aura but are isolated — no existing query touches them
unless `entry.concept` or `entry.phase` is non-null, which only happens for
stages that have the new edges. To remove SC010 from the graph too:

```cypher
MATCH (sc:Scenario {id: 'SC010'})-[:HAS_STAGE]->(st:Stage)
DETACH DELETE st;
MATCH (sc:Scenario {id: 'SC010'}) DETACH DELETE sc;
```

(The Concept and FrameworkPhase nodes from v25.0 are not touched by
this rollback — they remain available for future scenarios.)
